# People Knowledge Base Guidelines

Whenever working on a task involving a specific person:

- **Check** if they have an entry in `Knowledge/People/` (search all subdirectories).
- **Create** if missing -- at minimum: name, role/title, contact info, relationship context.
- **Update** if exists -- new topics, status changes, contact details from current interaction.
- **Categorize correctly** into the appropriate subdirectory.
- **Never duplicate.** One file per person. If someone changes category, move the file.
- **Cross-link** to relevant project files.

## People Entry Format

```markdown
# Full Name (nickname if applicable)

One-line description.

**Contact:** [method and details]
**Role:** [title]
**Organization:** [company or relationship]

## Background
[Bio, professional history, how you know them]

## Current Topics
[Active items involving this person, with project links]

## Related
[Cross-links to project files and other KB entries]
```

This file is yours to edit. Add fields specific to your needs (e.g., timezone, preferred communication method, pronouns), change the template structure, adjust the categorization rules. The instruction file just points here -- changing this file changes the behavior immediately.
